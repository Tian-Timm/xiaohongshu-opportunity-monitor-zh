# 小红书机会提醒 Skill

> 让 Agent 每天在小红书寻找新出现的实习、招聘、兼职或招募信息：自动扩展关键词、过滤过期内容、永久去重，并把无法确认的图片型帖子交给你核实。

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

适用于支持 Skills、可交互浏览器和定时任务的 Agent，例如 Codex。用户不需要懂自动化、脚本或数据库，只需填写一次搜索设置。

**快速开始** · [它怎样工作](#它怎样工作) · [怎样阅读结果](#怎样阅读结果) · [本地数据](#本地数据与隐私) · [项目结构](#项目结构)

## 它解决什么问题

小红书上的机会信息经常存在这些情况：

- 标题没有写完整岗位名，甚至使用缩写、谐音或公司代称；
- 具体要求藏在图片或聊天截图里；
- 同一帖子会被多个关键词反复搜到；
- 搜索结果混有旧帖、经验分享、课程和付费内推；
- 人工每天重复搜索很费时间。

这个 Skill 让 Agent 负责定时搜索、初筛和去重，把真正需要人判断的内容留给用户。

## 快速开始

### 1. 安装

#### 方法 A：直接让 Agent 安装（推荐给普通用户）

把下面整段话复制给支持 Skills 安装的 Agent：

```text
请帮我从下面的 GitHub 仓库安装这个 Skill：

https://github.com/Tian-Timm/xiaohongshu-opportunity-monitor-zh

Skill 位于：
skills/xiaohongshu-opportunity-monitor-zh

请安装到我的个人 Skills 目录。安装前检查 SKILL.md 和 scripts，
安装完成后告诉我是否成功。
```

Agent 可能会请求联网和写入个人 Skills 目录的权限，请确认授权范围后再继续。安装成功后，在下一轮对话中调用这个 Skill。如果 Agent 不具备 Skill 安装、联网或本地文件写入能力，请改用下面的方法。

#### 方法 B：使用 Skills CLI

```bash
npx -y skills add Tian-Timm/xiaohongshu-opportunity-monitor-zh -g --all
```

#### 方法 C：手动复制

把下面的文件夹复制到 Agent 的个人 Skills 目录：

```text
skills/xiaohongshu-opportunity-monitor-zh/
```

### 2. 检查运行条件

Agent 至少需要：

- 能操作真实网页的 Browser Use，或能控制桌面界面的 Computer Use；
- 可正常登录的小红书账号；
- 如果要每天自动运行，还需要定时任务能力。

首次运行时，建议用户在场完成登录、验证码和页面验证。Skill 不会保存或代填密码、验证码。

### 3. 开始设置

可靠的明确调用方式：

```text
使用 $xiaohongshu-opportunity-monitor-zh，帮我建立小红书机会提醒。
```

也可以直接说：

```text
每天帮我找小红书上新发布的 AI 产品实习，不要重复昨天的帖子。
```

在 Codex 中，启用的 Skill 会出现在斜杠列表里，也可以使用 `$xiaohongshu-opportunity-monitor-zh` 明确调用。Skill 还会识别 `/小红书机会` 和 `/xhs-monitor` 这两个文本快捷指令；这两个自定义别名是否会显示为独立菜单项，取决于所使用的 Agent 客户端。

### 4. 填写搜索设置

触发后，Agent 不会立刻自行决定关键词，而会请用户填写一张简单的表：

| 需要的信息 | 示例 |
|---|---|
| 想找什么机会 | AI 产品实习、游戏发行合作 |
| 地点或远程要求 | 北京或远程 |
| 必须满足的条件 | 日常实习、每周至少 4 天 |
| 不想看到的内容 | 培训、付费内推、销售岗 |
| 发布时间范围 | 默认最近 14 天 |
| 汇总时间 | 默认每天 21:00 |
| 已想到的关键词 | 可留空，由 Agent 生成 |

Agent 会根据回答生成搜索词和监测方案。用户回复“开始”后，才进行首次扫描并创建每日任务。

## 它怎样工作

```text
填写搜索设置
      ↓
根据目标生成核心词、长尾词和轮换词
      ↓
使用 Browser Use 或 Computer Use 操作小红书站内搜索
      ↓
过滤超过时间范围的帖子，并与历史记录永久去重
      ↓
将新帖子分为“已确认”和“请你核实”
      ↓
用户标记：值得看 / 不感兴趣 / 稍后再看
```

为减少重复操作和 Token 消耗，Skill 会：

- 每天使用少量核心搜索词和轮换搜索词；
- 在结果页只读取帖子 ID、标题、作者、时间和链接；
- 先过滤日期和历史重复，再打开真正的新候选；
- 不反复检查历史岗位是否更新；
- 不读取全部评论，也不对所有图片批量 OCR。

## 怎样阅读结果

每日报告只展示首次发现的新帖子。

| 标签 | 含义 | 你需要做什么 |
|---|---|---|
| ✅ 已确认 | 正文足以确认正在招聘或招募 | 查看要求，决定是否进一步了解 |
| 👀 请你核实 | 可能相关，但关键信息在图片、代称或模糊文本中 | 打开原帖，人工判断是否值得看 |

“请你核实”不代表低质量，只表示 Agent 无法在低成本分析下可靠确认。每条结果都会说明需要核实的原因，并保留原帖链接。

报告末尾会邀请用户直接回复：

```text
请回复我：哪些值得看、哪些你不感兴趣、哪些想稍后再看。
```

可以使用帖子标题、报告中的顺序或原帖链接，不需要记忆内部编号。

帖子一旦出现过，无论是否标记，都不会再次进入“今日新增”。用户可以随时要求查看“值得看”“不感兴趣”“稍后再看”或“尚未标记”的记录。

## 本地数据与隐私

每个监测任务的状态默认保存在当前项目中：

```text
work/xiaohongshu-monitor/<monitor-id>/state.json
```

其中包含见过的帖子、去重信息、搜索状态和用户标记。它不会保存小红书密码或验证码。

仓库已经通过 `.gitignore` 排除 `work/`，避免将用户的本地监测记录意外提交到 GitHub。

本 Skill 默认只读：不会自动私信、投递简历、上传附件、填写申请表或发送个人信息。所有对外操作都必须先获得用户明确确认。

## 已知限制

- 小红书的登录验证、风控和页面结构可能变化；
- 图片型帖子采用“扩大人工核实范围”的方式提高召回率，并不保证自动识别全部图片文字；
- 是否支持每日自动运行，取决于 Agent 是否提供定时任务能力；
- 启用的 Skill 可以出现在 Codex 的斜杠列表里，但 `/小红书机会` 这个自定义别名是否成为独立菜单项，取决于客户端；
- 本项目不是小红书官方产品，与小红书无隶属或合作关系。
